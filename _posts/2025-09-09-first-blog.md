### 첫 포스팅

- github 블로그를 첫 게시합니다.

feat: 새로운 기능 추가

fix: 버그 수정

docs: 문서 변경 (README, 주석 등)

style: 코드 포맷팅/세미콜론/스페이스 수정 (코드 동작 영향 없음)

refactor: 리팩토링 (동작은 동일, 내부 코드 구조 개선)

perf: 성능 개선

test: 테스트 관련 코드 추가/수정

build: 빌드 시스템, 의존성 관련 변경

ci: CI/CD 관련 설정 변경

chore: 기타 잡다한 변경 (예: 패키지 업데이트)

revert: 이전 커밋 되돌리기
